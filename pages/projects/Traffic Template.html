Project Details

How It Was Built

Developed in UE5 using EOS for peer-to-peer multiplayer. This project features a full asymmetrical horror gameplay loop.

This project is an asymmetrical horror prototype built in Unreal Engine 5 using a clean separation between visual logic (Blueprints/UI) and core gameplay systems (C++). The core loop is modeled after Dead by Daylight: survivors repair generators, power the exit, heal each other, and avoid being caught and hooked by the killer.

Most gameplay rules — interaction flow, survivor states (healthy/injured/crawling/carried/hooked), health progression, generator repair and sabotage, hook and wiggle logic, etc. — are implemented and replicated in C++ components. Those components drive authoritative game state on the server, and then push cosmetic/UI updates to clients. That includes survivor health bars, skill checks, action progress bars, and endgame triggers (survivors escape vs. killer wins).

The match flow runs on Epic Online Services (EOS). The server creates or joins EOS sessions, assigns one player as the killer and the rest as survivors, and replicates win conditions. All survivors are dead vs. the exit powered and survivors escaped.

Why

The goal of this project was to learn by recreating the full loop of an online asymmetrical horror game: networking, authority, session management, replicated combat, and interaction systems. I wanted to build something more complex than a “simple demo” and closer to an actual multiplayer game that I can experience with my friends.

Long-term, the goal is to turn this into a reusable asymmetric horror template. The plan is to replace any copyrighted placeholder content (like the Silent Hill: Midwich Elementary map and Pyramid Head killer) and ship a clean, original base that other devs can build on.

Built With

• Unreal Engine 5 (C++ gameplay framework + Blueprint-driven UI)
• Epic Online Services (sessions, lobbies, online identity, player presence)
• Niagara for hit / blood particle effects and generator VFX
• Custom movement components for both Survivor and Killer (sprint, crawl, stun states)
• Replicated animation montage system for melee, carry, hook, etc.
• Blender for modeling and kitbashing level content
• Retro PSX-style art direction (low poly, crunchy textures, heavy atmosphere)
• FL Studio for sound effects and audio work
