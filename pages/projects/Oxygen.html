Project Details

How It Was Built

Developed in Unreal Engine 5, this project is a narrative-driven audiovisual adaptation of Ivan Vyrypaev’s Oxygen, rebuilt as an interactive, stylized rave environment. The experience focuses on real-time audio reactivity, dynamic narration, and scene transformation tied directly to musical and emotional beats.

The core systems are built around a custom Audio-Reactive Environment Framework. Narration lines are triggered and sequenced through gameplay logic, synchronized with the music, and used to drive changes in lighting, post-processing, camera effects, and visual mood. Each narrative “pulse” alters the scene in real time, creating a constantly shifting aesthetic that mirrors the pacing of the text.

Real-time audio analysis and DSP (digital signal processing) are used to modify the music as the story progresses — adjusting intensity, filtering layers, or introducing transitions tied to narrative peaks. The environment responds dynamically, with lights, particle systems, and post effects updating instantly based on audio amplitude, frequency patterns, or scripted narrative cues.

All systems are implemented using a clear separation between logic and presentation. Core sequencing, timing, and audio-reactive calculations run in C++ components, while Blueprints drive the visual and cinematic responses. This keeps the system flexible for rapid iteration while ensuring tight synchronization across narration, music, and environmental effects

Why

The goal of this project was to explore how narrative and music can actively shape an environment in real time using only Unreal Engine’s native toolset. Instead of relying on external audio middleware, the challenge was to push Unreal’s built-in audio capabilities to create a fully reactive audiovisual space that shifts with the story.

By adapting Oxygen into a moving, music-driven environment, the project became a practical investigation into audio-driven design, dynamic scene mood transitions, and synchronized narration delivery. It serves as a complete standalone prototype — a space where music, voice, and visual style merge into a cohesive, reactive narrative experience.

Audio Manager

void AAudioManager::PlayerTriggeredNextLine()
{
	if (!NarrationLines.IsValidIndex(CurrentLineIndex)) return;

	// If already playing, allow exactly one queued trigger
	if (bIsPlaying)
	{
		bTriggerQueued = true;
		return;
	}

	// (inline) time-to-next-beat for everything after the first line
	const float Delay = (CurrentLineIndex == 0)
		? 0.001f
		: (BeatInterval - FMath::Fmod(GetWorld()->GetTimeSeconds() - StartTime, BeatInterval));

	FTimerHandle TH;
	GetWorld()->GetTimerManager().SetTimer(TH, [this]()
	{
		StartLine(CurrentLineIndex);
	}, Delay, false);
}

Light Snapshot Manager

void ALightSnapshotManager::Tick(float DeltaSeconds)
{
	Super::Tick(DeltaSeconds);

	if (!bBlending) return;

	BlendElapsed += DeltaSeconds;
	const float Alpha = FMath::Clamp(BlendElapsed / FMath::Max(BlendDuration, KINDA_SMALL_NUMBER), 0.f, 1.f);

	CurrentColor = FLinearColor::LerpUsingHSV(StartColor, TargetColor, Alpha);
	PushColorAll(CurrentColor);

	if (Alpha >= 1.f)
	{
		bBlending = false;
	}
}

void ALightSnapshotManager::ApplyLightSnapshot(EAudioSnapshot Snapshot, float BlendSeconds)
{
	if (const FLinearColor* InTargetColor = SnapshotColorTable.Find(Snapshot))
	{
		StartColor    = CurrentColor;
		TargetColor   = *InTargetColor;
		BlendDuration = FMath::Max(0.01f, BlendSeconds);
		BlendElapsed  = 0.f;
		bBlending     = true;

		if (BlendDuration <= 0.015f)
		{
			CurrentColor = TargetColor;
			PushColorAll(CurrentColor);
			bBlending = false;
		}
	}
}
